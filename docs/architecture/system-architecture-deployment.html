System Architecture Deployment Strategy

On-Premise Database + External Access
🏆 RECOMMENDED: Hybrid Cloud Architecture

Database on-premise (factory) + Backend & Frontend on Cloud (AWS/Azure) with VPN/Secure tunnel
📋 Component Deployment Guide

☁️ Cloud: Frontend

Location: AWS S3 + CloudFront (or Azure Blob + CDN)
What to deploy: Built React app (HTML/CSS/JS)
Why: Closer to users, better latency, auto-scaling
Cost: ~$1-5/month for small traffic

☁️ Cloud: Backend API

Location: AWS EC2 / ECS (or Azure App Service)
What to deploy: Node.js/Python API Server
Why: Can scale dynamically, easy to maintain
Cost: ~$20-50/month (t3.small instances)

🏭 Factory: Database

Location: Physical server in factory
What to deploy: PostgreSQL, InfluxDB, Redis
Why: Keep data on-premise, compliant, full control
Cost: One-time hardware + maintenance

🔒 Factory: VPN Gateway

Location: Physical/Virtual appliance in factory
What to deploy: OpenVPN server or IPSec gateway
Why: Encrypted tunnel between Cloud & Factory
Cost: ~$5-10/month for OpenVPN Cloud
🔐 Security Flow:

User (Internet) → HTTPS to CDN/API in Cloud → Cloud Backend (EC2) → VPN Encrypted Tunnel → Factory VPN Gateway → Internal Private Network → Database (192.168.1.10)
⚠️ Important Security Measures:

• Database has NO direct internet access (only via VPN)
• Firewall on factory network allows only port 443 (HTTPS)
• Database uses private IP (192.168.x.x)
• VPN tunnel is encrypted (IPSec/OpenVPN)
• Backend API validates JWT tokens before DB access
• Database has read-only API access configured
• Regular backups to cloud or external disk

Option 2: Hybrid Cloud

Setup: Frontend & Backend on Cloud, DB on-premise
Frontend: S3 + CloudFront
Backend: EC2 / App Service
Database: PostgreSQL in factory
Connection: VPN Tunnel (IPSec/OpenVPN)
✓ Pros
Best scalability
Easy external access
Data stays on-premise
Secure (encrypted tunnel)
✗ Cons
Cloud costs (~$30/month)
Network latency (100-200ms)
VPN maintenance
Moderate complexity

Option 3: Cloud + Backup DB

Setup: Replicate DB to cloud, read-only access
Frontend: Cloud (S3 + CDN)
Backend: Cloud with cloud DB
Database: Master on-premise, Replica in cloud
Sync: Continuous replication
✓ Pros
Fast cloud queries
Good scalability
Better disaster recovery
Data local backup
✗ Cons
Higher cloud costs
Data sync complexity
Eventual consistency
Replication lag

Option 4: Hybrid with Edge

Setup: Cloud frontend, local backend, on-premise DB
Frontend: S3 + CloudFront
Backend: Node.js in factory (VPN + API Gateway)
Database: PostgreSQL in factory
Access: AWS API Gateway with IAM auth
✓ Pros
Very low latency
Full control (backend)
Data stays local
Best performance
✗ Cons
Backend maintenance needed
No cloud scaling
Network dependency
Complex setup

✅ Why Hybrid Cloud is Recommended:

• Best balance between cost ($30-50/month), security, and scalability
• Database stays on-premise (data control + compliance)
• External users get fast, reliable access via cloud
• Encrypted VPN tunnel between cloud and factory
• Easy to scale backend as users grow
• Minimal maintenance compared to on-premise only
🛠️ Implementation Steps (Hybrid Cloud)
1

Prepare Factory Infrastructure

Timeline: Week 1-2
• Get physical server for DB (or repurpose existing)
• Install PostgreSQL + InfluxDB
• Configure internal network (192.168.1.x)
• Setup local backup system

2

Setup VPN Gateway (Factory)

Timeline: Week 2-3
• Buy OpenVPN cloud license (~$5/month)
• Or: Cisco/Fortinet VPN box
• Configure IPSec/OpenVPN tunnel
• Setup firewall rules (port 443 only)

3

Setup AWS/Azure Cloud Account

Timeline: Day 1
• Create AWS/Azure account
• Setup VPC with security groups
• Create S3 bucket for frontend
• Launch EC2/AppService instance

4

Connect VPN (Cloud ↔ Factory)

Timeline: Week 3
• Install VPN client on cloud backend
• Configure VPN server in factory
• Test encrypted tunnel
• Verify database connectivity

5

Deploy Backend API (Cloud)

Timeline: Week 4
• Upload Node.js/Python app to EC2
• Configure environment variables
• Setup SSL certificates
• Test API endpoints

6

Deploy Frontend (Cloud CDN)

Timeline: Week 4
• Build React app: npm run build
• Upload to S3 bucket
• Configure CloudFront distribution
• Point domain to CDN

7

Security Hardening

Timeline: Week 5
• Configure IAM roles
• Setup JWT authentication
• Enable HTTPS everywhere
• Test security penetration

8

Testing & Go Live

Timeline: Week 5-6
• Test from external network
• Load testing
• User acceptance testing
• Launch to production

📊 Network Diagram Details
FACTORY NETWORK:
• Firewall: Only port 443 (HTTPS)
• Gateway: 192.168.1.1 (OpenVPN server)
• DB Server: 192.168.1.10 (PostgreSQL/InfluxDB)
• Internal LAN: 192.168.1.0/24 (PRIVATE)
• No direct internet access to DB
CLOUD BACKEND (AWS EC2):
• Public IP: Assigned by AWS
• Security Group: Allow 443 incoming
• VPN Client: Connected to factory VPN
• Can reach DB via: 192.168.1.10 (via tunnel)
• API Endpoint: https://api.mes.com/v1/...
CDN (CloudFront):
• Caches: React app (HTML/CSS/JS)
• Global distribution (faster for users)
• Points to S3 bucket as origin
• Domain: https://dashboard.mes.com
💡 Data Flow (User Request):

1. User opens dashboard.mes.com
2. Browser requests from CloudFront CDN
3. CDN returns React app (HTML/CSS/JS)
4. React app loads and calls api.mes.com/v1/dashboard
5. Cloud Backend receives request
6. Backend connects via VPN tunnel to Factory
7. Backend queries PostgreSQL (192.168.1.10)
8. Database returns data
9. Backend sends JSON response
10. React app displays data in Dashboard

⏱️ Total time: 200-500ms (depending on latency)
⚠️ Important Considerations:

• Firewall in factory: Must whitelist VPN gateway IP
• Database password: Store in cloud secret manager (AWS Secrets)
• Network bandwidth: Ensure factory internet can handle traffic
• Backup strategy: Regular backups to cloud (S3 or Glacier)
• Monitoring: Setup CloudWatch + local monitoring
• Redundancy: Consider multi-AZ for cloud resources
